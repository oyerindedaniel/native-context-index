# What &ldquo;symbol&rdquo; means in NCI

<PageLede>
  A symbol is one merged graph row — one declaration in the model the agent queries. Classes own their members. Overloads can either fuse into one row or stay as separate rows, depending on how their normalised signatures collide. The columns below are the public ones you can read with `nci query` and `nci sql`.
</PageLede>

## Path encoding (`__nci_external__`, `__up__`)

When a declaration resolves outside the indexed package root, `file_path` does **not** use raw `../` segments. Instead it uses canonical tokens — `__nci_external__` plus one `__up__` segment per parent hop. Treat those as **stable SQLite keys**, not filesystem paths.

For real disk access, combine `source_package_name` + `source_file_path` with the install layout. If that still does not yield a real file, fall back to `nci query snippet` / `nci query show` — those return cite-ready text without needing a path.

## Three id shapes

<CalloutRoot variant="info">
  <CalloutTitle>Pick the right id for the join.</CalloutTitle>
  <CalloutBody>
    <code>symbols.symbol_id</code> (int) → joins to <code>symbol_dependencies.from_symbol_id</code>, <code>symbol_inherited_from_sources.symbol_id</code>, etc. <code>symbols.id</code> (string) → handles for <code>query show</code> / <code>snippet</code> / <code>overloads</code> and target of <code>symbol_dependencies.to_symbol_id_text</code>. <code>id</code> suffixes <code>#2</code>, <code>#3</code>, … disambiguate homonyms — different rows, not overload indices on their own.
  </CalloutBody>
</CalloutRoot>

## Worked example: a class&rsquo;s members

<TerminalRoot>
  <TerminalCommand>nci query symbol ZodObject --package zod -n 1 --format json</TerminalCommand>
</TerminalRoot>

Take the `id` from that JSON envelope and feed it into `query overloads` to see every sibling row, or join in SQL on `parent_symbol_id` to walk a class&rsquo;s declared members.

```sql
SELECT m.name, m.signature
FROM symbols m
JOIN symbols c ON c.symbol_id = m.parent_symbol_id
JOIN packages p ON p.package_id = c.package_id
WHERE c.name = 'ZodObject'
  AND c.kind_name = 'ClassDeclaration'
  AND p.name = 'zod'
ORDER BY m.name;
```
