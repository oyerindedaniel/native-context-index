# NCI skills

<PageLede>
  Skills are short SKILL.md files that tell an agent how to think when it touches NCI. They live under `skills/` in the repo and ship as a single shared knowledge layer for any client that loads them.
</PageLede>

## What a skill is

A skill is a plain Markdown file with frontmatter. The frontmatter names the skill and decides whether the host can auto-invoke it; the body is a short protocol the agent follows when the skill is in scope.

```
skills/
  README.md
  <skill-name>/
    SKILL.md
```

Hosts that understand the open [Agent Skills](https://agentskills.io/) shape each discover `SKILL.md` from their own paths — there is no single global folder. The table below is a map; check each product&rsquo;s docs when they ship an update.

Install from git with <a href="https://github.com/vercel-labs/skills" target="_blank" rel="noopener noreferrer">npx skills</a>:

<TerminalRoot>
  <TerminalCommand>npx skills add oyerindedaniel/native-context-index --skill nci --skill nci-answer-quality -y</TerminalCommand>
</TerminalRoot>

`nci-answer-quality` sets `disable-model-invocation: true` so the protocol is not paraphrased when it fires. `nci` leaves invocation flexible for onboarding judgment.

<CalloutRoot variant="info">
  <CalloutTitle>Primer stays in sync.</CalloutTitle>
  <CalloutBody>
    <code>skills/nci/PRIMER.md</code> is generated from the same package that powers <code>nci://primer/agent</code> and the <a href="/docs/primer">Agent primer</a> compact tab. After editing <code>@repo/nci-agent-primer</code>, run <code>pnpm sync:skill-primer</code> from the monorepo root.
  </CalloutBody>
</CalloutRoot>

The skills directory is not part of the CLI surface — `nci` does not read or write it. It is a sibling to NCI that any agent runtime can mount.
