# MCP method registry

<PageLede>
  `nci-mcp` registers exactly two tools and three resources. There are no other methods, no admin tools, no hidden helpers — the table below is the complete surface.
</PageLede>

<CalloutRoot variant="warning">
  <CalloutTitle>Don&rsquo;t act on the database path.</CalloutTitle>
  <CalloutBody>
    The path returned by <code>nci://database/active</code> is for diagnosis only. Do not delete, vacuum, reindex, or modify it from the agent loop — `nci_query` and `nci_sql` already operate against the same database without needing it. If a tool call appears to use the wrong DB, pass <code>database</code> on the next call instead of mutating files.
  </CalloutBody>
</CalloutRoot>

## Server identity

The MCP server announces itself as `name: "nci-mcp"` with a version that matches its npm package version. The server&rsquo;s instructions string (set in `packages/nci-mcp/src/index.ts`) tells the agent to read `nci://primer/agent` first — the same advice rendered at [Agent primer](/docs/primer).
